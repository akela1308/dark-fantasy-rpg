## Что это за проект

Тактическая RPG в жанре тёмного фэнтези, вдохновлённая **Disciples II**.
- Стек: **Phaser 3.80 + Vite 5**, чистый ES6
- Репозиторий: `https://github.com/akela1308/dark-fantasy-rpg`
- Разработчик: Максим (акаунт akela1308)
- Путь на маке: `/Users/maksimilin/Desktop/Dark Fantasy/dark-fantasy-rpg/`
- Запуск: `npm run dev` → `http://localhost:5173`
- Билд: `npm run build` → папка `dist/`

    TurnManager.js      — очередь ходов по speed DESC, PHASE state machine

    WalkableZones.js    — зоны хождения по карте (прямоугольники в world coords)

public/
  loading.png           — арт на экране загрузки
  battle_bg.png         — фон боевой сцены
  audio/                — MP3 треки (7 штук)
  maps/                 — map1.png, tavern_map.png и др.

## Поток сцен

```
BootScene  →  LoadingScene  →  MapScene  →  (encounter)  →  LoadingScene  →  BattleScene
                                    ↑                                              |
                                    └─────────── [ На карту ] (победа) ───────────┘
```

- **BootScene**: логотип + кнопка НАЧАТЬ
- **LoadingScene**: грузит ВСЕ ассеты (спрайты, карты, аудио). Запускает первый музыкальный трек как только он загружен. Переход через `load.on('complete')`. `create()` пустой (переход уже в 'complete').
- **MapScene**: карта мира. Герой ходит по клику. Два компаньона следуют по trail. Бандит патрулирует дорогу. При дистанции < 90px → fade → LoadingScene → BattleScene.
- **BattleScene**: тактический бой. После победы → флаг `bandit_0_defeated` в registry → кнопки "Попробовать снова" и "На карту".

## Система боя

### TurnManager
- Сортировка по `speed` DESC → очередь ходов
- Состояния (PHASE): `player_input` → `ai_thinking` → `resolving` → `end`
- `nextTurn()`: tick cooldowns + effects у закончившего ход юнита, переход к следующему живому
- `skipTurn()`: emit log + nextTurn

### BattleGrid
- Логическая сетка 2 ряда × 3 колонки для каждой стороны
- Ближний бой: задний ряд (row=1) недоступен если в переднем (row=0) есть живые
- Дальнобойные (ranged=true) бьют кого угодно

### AI (EnemyUnit.decideAction)
- Цель: слабейший живой юнит в досягаемости
- Передний ряд приоритетнее заднего
- Мини-босс при HP < 50% кричит `commanders_roar` (buff всей стороны)

### Эффекты (Unit.effects[])
- `dodge_boost`: шанс уклонения при получении урона
- `covered_by`: следующий удар принимает указанный юнит вместо текущего

### SkillSystem
- JSON скиллы: строчный `effect` ("dodge_boost", "cover") → `_applyBuiltin()`
- Функциональные эффекты (commanders_roar) → `typeof skill.effect === 'function'`

## Карта мира (MapScene)

### Система координат
- Карта: 1672×941 пикселей (мировые координаты)
- Canvas: 1280×720
- Zoom: `Math.min(1280/1672, 720/941)` ≈ 0.765
- `cameras.main.setZoom(zoom).centerOn(836, 470)`
- Клики: `ptr.worldX / ptr.worldY` (НЕ ptr.x/y!)
- Все объекты (юниты, зоны) в мировых координатах

### Движение отряда
- Герой: moveTo(clamped walkable position)
- Боец: `_followTrail(brawler, 15)` — 15 точек позади
- Знахарка: `_followTrail(healer, 28)` — 28 точек позади
- `_separateParty()`: вызывается каждый кадр, выталкивает компаньонов если ближе 55px друг к другу (герой приоритет 1, боец 2, знахарка 3)

### WalkableZones (map1)
Дорога идёт диагонально: левый-низ → правый-верх
```
{ x: 0,    y: 570, w: 220,  h: 200 }
{ x: 170,  y: 530, w: 260,  h: 200 }
{ x: 390,  y: 490, w: 300,  h: 195 }
{ x: 650,  y: 450, w: 350,  h: 195 }
{ x: 960,  y: 410, w: 380,  h: 195 }
{ x: 1290, y: 370, w: 382,  h: 200 }
```
`clamp(x, y)` — если вне зоны, находит ближайшую точку внутри

### Бандит
- Один patrol по waypoints: (750,510) → (950,470) → (840,530)
- Если `game.registry.get('bandit_0_defeated')` === true → не спавнится (после победы)

## Музыка

- Запускается в LoadingScene при загрузке первого трека (`track_ashes2`)
- Обход autoplay: `if (this.sound.locked) { this.sound.once('unlocked', startMusic) }`

- Треки: Ashes of Velanth 2, Ashes of Velanth, Ashes of the Last Ward, Ashes of the Monastery 2, Ashes of the Monastery, Ashes of the Pass, Dark Song
- MusicPlayer: мини-панель сверху-слева (♪ ◀ название ▶), destroy старого при переключении

## EventBus (глобальная шина событий)

```
log               — строка → UIManager.appendLog
turn_started      — unit → UIManager.highlightActive
unit_damaged      — {unit, amount} → animHit + animDamageNumber
unit_healed       — {unit} → animHeal
unit_died         — unit → animDeath
skill_used        — {caster, skillId, target}
skill_selected    — skillId → BattleScene._onSkillSelect
skip_turn         — () → TurnManager.skipTurn + _renderAll
commanders_roar   — caster → (buff всех врагов, не реализован полностью)
xp_gained         — {unit, amount}
level_up          — {unit, choice}
```

`eventBus.clear()` вызывается в начале каждого `BattleScene.create()` — это важно, иначе события накапливаются.

## Рендер боя (BattleScene._renderAll)

Вызывается после каждого хода (в т.ч. после skip_turn). Полностью пересоздаёт спрайты:
- Уничтожает все `_unitSprites`
- Для каждого живого юнита: позиция из `UNIT_POSITIONS[side][row][col]`
- Глубина: row=0 (передний) → depth 3, row=1 (задний) → depth 2
- Спрайты с `setOrigin(0.5, 1)` — стоят "ногами" на точке позиции
- Враги: `setFlipX(true)`. Дуэлянт: `setFlipX(true)` (лицом к врагам)
- Кольцо под ногами: эллипс с мерцающим tweens
- Интерактивность: враги → `_onEnemyClick`, союзники → `_onAllyClick`

Позиции юнитов (пиксели экрана):
```
player:
  row=0: col0=(470,470) col1=(320,490) col2=(580,450)
  row=1: col0=(390,320) col1=(250,340) col2=(500,300)
enemy:
  row=0: col0=(760,480) col1=(920,455) col2=(650,460)
  row=1: col0=(820,330) col1=(1000,310) col2=(700,315)
```

Высота спрайта: row=0 → 230px, row=1 → 160px (перспектива Disciples II)

## PortraitPanel

- Слева: игровые юниты (портреты + HP-бар + цифры)

- Карточка: 96×130px, отступ от края 4px

## Ключевые баги которые были исправлены

1. **SkillSystem crash**: `skill.effect` была строкой "dodge_boost" но вызывалась как функция → `typeof === 'function'` guard
2. **Grid sync**: мёртвые юниты не убирались с грида если tween.onComplete не срабатывал → синхронный loop после nextTurn
3. **Cover эффект**: перехват урона работал только если cover проверялся до dodge → переставлен в начало takeDamage
4. **LoadingScene double start**: create() вызывал scene.start() дублируя 'complete' handler → убран из create()
5. **Memory leak музыки**: old sound.stop() без destroy() → добавлен destroy()
6. **UIManager hint**: getByName('hint') нестабилен → прямая ссылка на объект
7. **Аудио autoplay**: браузер блокирует → sound.locked + sound.once('unlocked')
8. **Карта координаты**: ptr.x/y вместо ptr.worldX/Y → неправильный клик; исправлено
9. **Walk animation y-conflict**: MapUnit bob tween двигал y пока movement тоже двигал y → убран y из tween, только angle
10. **Separation**: компаньоны наслаивались на героя → `_separateParty()` каждый кадр

## Технические заметки

- `Phaser.Scale.FIT + CENTER_BOTH` — игра масштабируется под экран сохраняя 16:9
- `game.registry` — глобальное хранилище между сценами (bgMusic, bgMusicIndex, musicStarted, bandit_0_defeated)
- `eventBus` — Phaser.Events.EventEmitter singleton, импортируется везде
- В `BattleScene.create()` первым делом `eventBus.clear()` — обязательно!
- Спрайты в `public/sprites/`, портреты в `public/portraits/`, карты-персонажи в `public/maps/characters/`
- JSON данные в `src/data/` — импортируются как ES modules напрямую
- Vite разрешает `import json from './file.json'` нативно
